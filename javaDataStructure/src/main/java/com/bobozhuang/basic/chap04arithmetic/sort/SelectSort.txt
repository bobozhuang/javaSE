2、优化选择排序
    将下标 1、2对比，下一次小for循环 2、3对比，每次如果小于只会交换位置，小for循环执行完只有9换到了最后的位置（只进行了一次
    交换），相比较之前最优要少交换7次，大for循环控制下一次循环开始，这样九个数字只进行了9-1次循环

4、选择排序的时间复杂度：简单选择排序的比较次数与序列的初始排序无关。
    比较次数：
        假设待排序的序列有 N 个元素，则比较次数永远都是N (N - 1) / 2。
    移动次数：
        移动次数与序列的初始排序有关。当序列正序时，移动次数最少，为 0。
        当序列反序时，移动次数最多，为3N (N - 1) /  2。(我认为是(N-1)/2)
    所以，综上，简单排序的时间复杂度为 O(N2)。这个是N的2次方

    证明：N个元素比较次数为N (N - 1) / 2
    第一个元素与之后的元素逐个进行比较一共比较(N - 1)次，比较完之后可以得到最小的元素，所以下一轮的比较从第二个元素开始，
    第二个元素与之后的元素逐个进行比较一共比较(N - 2)次，依此类推最后只比较一次：
    (N - 1)、(N - 2)、....1
    可以看出这是一个等差数列，等差数列求和公式Sn=n*(a1+an)/2，a1为第一个元素，an为之后一个元素，n为元素的个数
    所以比较次数为( N - 1 ) * ( N - 1 + 1 ) / 2 = N (N - 1) / 2。
